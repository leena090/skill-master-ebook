MCP 도구 확장 — AI의 손과 발

MCP(Model Context Protocol)는 AI에게 외부 도구를 연결해주는 기술입니다.
마치 스마트폰에 앱을 설치하듯, AI에게 새로운 능력을 추가합니다.

📧

Gmail

이메일 자동 작성, 검색, 답장

📅

Google Calendar

일정 생성, 빈 시간 찾기

📋

Notion

문서 작성, DB 관리, 보고서 저장

🌐

Playwright

웹 브라우저 자동 조작, SNS 게시

📝

WordPress

블로그 글 작성, 미디어 업로드

🔍

Web Search

실시간 인터넷 검색, 최신 정보

MCP = AI의 앱스토어

MCP 도구를 연결할수록 AI가 할 수 있는 일이 늘어납니다.
이메일 보내기, 일정 잡기, 블로그 발행, SNS 게시 — 전부 AI가 직접 처리합니다.

미래 비전 — 한 문장이면 사업이 돌아간다

스킬이 충분히 진화하면, 여러분이 할 일은 딱 한 줄 입력뿐입니다.
나머지는 AI 팀이 알아서 처리합니다.

입력한 것:

"매물접수 힐스테이트영통 전세 5.8억"

자동으로 생산된 것:

17
파일 생성
7
종 산출물
2
플랫폼 발행
5분
소요 시간

이것이 스킬의 최종 목표입니다.
여러분은 방향만 지시하고, AI가 실행합니다.
당신은 사장님이고, AI는 당신의 팀입니다.

클로드에게 주문하는 법 — 프롬프트 팁

같은 AI라도 어떻게 말하느냐에 따라 결과가 완전히 달라집니다.

이렇게 하지 마세요
# 너무 모호한 요청
"블로그 써줘"

# 맥락 없는 요청
"좋은 글 만들어"

# 한꺼번에 너무 많은 요청
"블로그도 쓰고 이미지도 만들고
SNS도 올리고 메일도 보내줘"

모호하면 AI도 모호하게 답합니다.
한꺼번에 너무 많으면 품질이 떨어집니다.

이렇게 하세요
# 구체적인 요청
"매물접수 힐스테이트영통 전세 5.8억"

# 맥락 + 기준 포함
"블로그 써줘. 주제: AI 자동화.
 대상: 50대 초보. 2000자. 친근한 톤."

# 단계별 요청
"먼저 설계부터 해줘.
 plan.md 보여주고, 내가 승인하면 실행."

구체적 + 맥락 포함 + 단계별
이 3가지만 지키면 결과가 확 달라집니다.

프로젝트 폴더 세팅 — 처음 시작

새 프로젝트를 시작할 때, 아래 순서대로 따라하면 됩니다.

  1. 프로젝트 폴더 만들기
    mkdir ~/Desktop/my-project
    바탕화면에 프로젝트 폴더 생성
  2. Claude 설정 폴더 만들기
    mkdir -p ~/.claude/skills ~/.claude/commands ~/.claude/agents
    스킬, 명령어, 에이전트 폴더 한 번에 생성
  3. CLAUDE.md 생성하기
    Claude Code에게 "CLAUDE.md 만들어줘"라고 말하면 됩니다
  4. 첫 스킬 파일 생성하기
    CH3에서 배운 대로, skills/ 폴더에 .md 파일 생성
  5. "설계부터 해줘" 로 시작!
    플랜 모드로 시작 → 승인 → 실행의 안전한 루틴
처음에 이것만 기억하세요

완벽하게 세팅할 필요 없습니다. CLAUDE.md 하나만 있어도 시작할 수 있습니다.
나머지는 사용하면서 하나씩 추가하면 됩니다.

CLAUDE.md 구성 가이드

CLAUDE.md에 꼭 들어가야 할 4가지 필수 섹션입니다. 

# CLAUDE.md — 필수 섹션 4가지

## 1. Hard Rules (절대 규칙)
- plan.md 승인 전 코드 작성 금지
- API 키 하드코딩 금지
- 한국어 주석 필수
- 작업 완료 후 노션 보고서 업로드

## 2. Design Standard (디자인 기준)
- "내가 돈 주고 사겠는가?" 기준
- 텍스트만 나열하면 재작업
- 50대 이상 사용자 기준 가독성

## 3. Security (보안 규칙)
- API 키는 반드시 .env 파일 사용
- 민감 정보 GitHub 커밋 금지
- 키 변경 시 즉시 보고

## 4. Agent Modes (모드 설정)
- "설계" → /architect 모드 전환
- "검토" → /critic 모드 전환
- "QA" → 자체 품질 검증 실행

Hard Rules

"이건 무조건 지켜라"
가장 중요합니다. 이것부터 쓰세요.
예: 승인 없이 실행 금지

Design Standard

"이 수준 이하면 다시 해라"
결과물 품질의 최소 기준선입니다.
예: 구매자 시각 기준 평가

Security

"비밀은 지켜라"
API 키, 비밀번호 유출 방지 규칙입니다.
예: .env 파일로 키 관리

Agent Modes

"이 말 하면 이 모드로 전환해라"
키워드별 동작 모드를 지정합니다.
예: "설계" → plan.md 작성
