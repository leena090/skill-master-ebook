Step 1: 폴더 구조 만들기

AI에게 업무를 가르치려면, 먼저 "교재를 놓을 책장"이 필요합니다.
아래 3개 폴더만 만들면 준비 끝입니다. 

# 홈 디렉토리에 .claude 폴더 생성
~/.claude/
├── skills/          # 스킬 파일 저장소 (업무 매뉴얼)
│   ├── blog-auto.md
│   ├── report-gen.md
│   └── sns-publish.md
│
├── commands/        # 슬래시 명령어 (/매물접수 같은 단축키)
│   └── 매물접수.md
│
├── agents/          # 에이전트 설정 (AI 팀원 역할 정의)
│   ├── researcher.md
│   └── writer.md
│
└── CLAUDE.md        # 총 설정 파일 (AI의 행동 규칙서)
왜 폴더를 나눌까요?

서류를 종류별로 파일철에 넣듯, AI도 정리된 구조에서 더 똑똑하게 일합니다.
skills = 업무 매뉴얼, commands = 단축키, agents = 팀원 이력서

Step 2: CLAUDE.md — AI의 행동 규칙서

CLAUDE.md는 AI에게 "이렇게 일해라"고 알려주는 사규집입니다. 

# CLAUDE.md 기본 구조

## Hard Rules (절대 규칙)
- 코드 작성 전 반드시 plan.md 승인
- API 키 절대 하드코딩 금지
- 한국어 주석 필수

## Design Standard (디자인 기준)
- "내가 돈 주고 사겠는가?" 기준
- 시각 요소 없는 텍스트 나열 = 재작업

## Agent Modes (모드 설정)
- "설계" → /architect 모드
- "검토" → /critic 모드

## Security (보안)
- .env 파일로 키 관리
- 민감 정보 커밋 금지
Hard Rules

"절대 어기면 안 되는 것"을 적습니다.
신입 직원에게 첫날 알려주는 것과 같습니다.

Design Standard

결과물의 최소 품질 기준을 정합니다.
이걸 정해두면 AI가 알아서 수준을 맞춥니다.

Security

비밀번호나 API 키가 외부에 노출되지 않도록
보안 규칙을 반드시 포함하세요!

Step 4: 스킬 파일 작성법

스킬 파일은 AI에게 주는 업무 매뉴얼입니다. 4가지 섹션만 채우면 됩니다. 

# 스킬 파일 예시: blog-auto.md

## 트리거 (언제 실행?)
키워드: "블로그", "포스팅"
→ 이 단어가 나오면 자동 실행

## 단계 (어떻게 진행?)
1. 주제 키워드 분석
2. SEO 제목 3개 생성
3. 본문 2000자 작성
4. 워드프레스 발행

## 에이전트 (누가 담당?)
- researcher: 자료 조사
- writer: 글 작성
- publisher: 발행 담당

## 산출물 (결과는?)
- 블로그 포스트 1건
- 썸네일 이미지 1장
- SNS 홍보문구 3건

Step 5: 테스트와 반복

스킬을 만들었으면 바로 실행해보세요.
처음부터 완벽할 필요 없습니다. 고치면서 완성하면 됩니다.

  1. 실행하기 — 스킬 트리거 키워드를 입력합니다.
    예: "블로그 써줘" → 블로그 자동화 스킬 실행
  2. 결과 확인하기 — 산출물이 예상대로 나왔는지 봅니다.
    예: "제목이 너무 딱딱해" → 피드백 메모
  3. 수정하기 — 문제점을 스킬 파일에 반영합니다.
    예: "제목은 친근한 말투로" 규칙 추가
이것이 "피드백 루프"입니다

실행 → 확인 → 수정을 반복할수록 스킬이 점점 똑똑해집니다.
마치 신입 직원이 경험을 쌓으며 성장하는 것과 같습니다.

Step 6: Hooks와 Memory

스킬을 더 똑똑하게 만드는 두 가지 비밀 무기가 있습니다.

Hooks — 자동 반응

"이런 상황이 생기면 자동으로 이걸 해라"

  • 메시지 제출 시 → 스킬 트리거 자동 감지
  • 파일 저장 시 → 자동 포맷팅
  • 에러 발생 시 → 자동 복구 시도

비유: 현관문 센서등 — 사람이 오면 자동 점등

Memory — 장기 기억

"이전에 배운 걸 다음에도 기억해"

  • 사용자 선호도 기억 (말투, 스타일)
  • 실수했던 패턴 기억 (같은 실수 방지)
  • 프로젝트 맥락 기억 (연속 작업 가능)

비유: 단골 카페 — "평소 드시는 거 드릴까요?"

SendMessage — 에이전트 간 소통

에이전트들은 SendMessage를 통해 서로 대화합니다.
마치 회사 메신저로 업무를 주고받는 것과 같습니다. 

# 리서처 → 라이터에게 조사 결과 전달
SendMessage(
  to: "writer",
  message: "시세 조사 완료!
    - 전세 시세: 5.2~5.8억
    - 최근 거래: 5.5억 (3개월 전)
    - 주변 시세: 4.8~6.0억"
)

# 라이터 → 퍼블리셔에게 원고 전달
SendMessage(
  to: "publisher",
  message: "블로그 원고 작성 완료.
    파일: output/blog-post.html
    발행 부탁드립니다."
)
에이전트끼리 알아서 대화합니다

여러분이 할 일은 처음에 "매물접수 해줘"라고 한 마디 하는 것뿐.
나머지는 에이전트들이 서로 메시지를 주고받으며 처리합니다.

검증 게이트 — 단계별 품질 체크

파이프라인 중간중간에 "검문소"를 설치합니다.
품질 기준을 통과해야만 다음 단계로 넘어갑니다.

왜 게이트가 중요할까요?

검증 없이 바로 발행하면 오류가 그대로 공개됩니다.
게이트 = 품질 보증 장치. 불량품이 출하되지 않도록 막아줍니다.

실전 체인: 매물접수 전체 타임라인

"매물접수 힐스테이트영통 전세 5.8억" 한 줄 입력 후 일어나는 일:

Step 1 — 트리거 감지
"매물접수" 키워드 → 스킬 자동 실행, 매물 정보 파싱
Step 2 — 시세 조사 (리서처)
네이버 부동산 기반 시세 분석, 주변 매물 비교 리포트 생성
Step 3 — 카피 작성 (라이터)
블로그 본문 + SNS 홍보문구 + 매물 요약 카드 작성
Step 4 — 산출물 생성 (디자이너)
엑셀 시세표, PPT 매물장, 카드뉴스 이미지 생성
Step 5 — 검증 게이트
모든 산출물 품질 확인 — 오류 시 자동 수정
Step 6 — 발행 (퍼블리셔)
워드프레스 블로그 발행 + 인스타그램 자동 게시 완료!
입력 1줄 → 7종 산출물 + 2개 플랫폼 자동 발행 = 약 5분

나만의 체인 설계하기

아래 워크시트를 채워보세요. 여러분만의 자동화 파이프라인이 완성됩니다.

나의 체인 설계 워크시트

1. 입력 (Input)

예: "블로그 써줘 [주제]"

2. 에이전트 1 (조사)

예: 리서처 — 자료 수집

3. 에이전트 2 (제작)

예: 라이터 — 글 작성

4. 산출물 (Outputs)

예: 블로그 1건 + 이미지 1장
처음에는 2~3단계로 시작하세요

복잡한 체인은 나중에 만들면 됩니다. 입력 → 가공 → 출력, 이 3단계면 충분합니다.
잘 돌아가면 그때 단계를 추가하세요.
