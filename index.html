- 코드 작성 전 반드시 plan.md 승인

- 한국어 주석 필수

## Design Standard (디자인 기준)
- "내가 돈 주고 사겠는가?" 기준
- 시각 요소 없는 텍스트 나열 = 재작업

## Agent Modes (모드 설정)
- "설계" → /architect 모드
- "검토" → /critic 모드

## Security (보안)
- .env 파일로 키 관리
- 민감 정보 커밋 금지
Hard Rules

Step 5: 테스트와 반복

스킬을 만들었으면 바로 실행해보세요.
처음부터 완벽할 필요 없습니다. 고치면서 완성하면 됩니다.

  1. 실행하기 — 스킬 트리거 키워드를 입력합니다.

Step 6: Hooks와 Memory

스킬을 더 똑똑하게 만드는 두 가지 비밀 무기가 있습니다.

Hooks — 자동 반응

"이런 상황이 생기면 자동으로 이걸 해라"

  • 메시지 제출 시 → 스킬 트리거 자동 감지
  • 파일 저장 시 → 자동 포맷팅
  • 에러 발생 시 → 자동 복구 시도

비유: 현관문 센서등 — 사람이 오면 자동 점등

Memory — 장기 기억

"이전에 배운 걸 다음에도 기억해"

  • 사용자 선호도 기억 (말투, 스타일)
  • 실수했던 패턴 기억 (같은 실수 방지)
  • 프로젝트 맥락 기억 (연속 작업 가능)

비유: 단골 카페 — "평소 드시는 거 드릴까요?"

SendMessage — 에이전트 간 소통

에이전트들은 SendMessage를 통해 서로 대화합니다.
마치 회사 메신저로 업무를 주고받는 것과 같습니다. 

# 리서처 → 라이터에게 조사 결과 전달
SendMessage(
  to: "writer",
  message: "시세 조사 완료!
    - 전세 시세: 5.2~5.8억
    - 최근 거래: 5.5억 (3개월 전)
    - 주변 시세: 4.8~6.0억"
)

# 라이터 → 퍼블리셔에게 원고 전달
SendMessage(
  to: "publisher",
  message: "블로그 원고 작성 완료.
    파일: output/blog-post.html
    발행 부탁드립니다."
)

실전 체인: 매물접수 전체 타임라인

"매물접수 힐스테이트영통 전세 5.8억" 한 줄 입력 후 일어나는 일:

Step 1 — 트리거 감지
"매물접수" 키워드 → 스킬 자동 실행, 매물 정보 파싱
Step 2 — 시세 조사 (리서처)
네이버 부동산 기반 시세 분석, 주변 매물 비교 리포트 생성
Step 3 — 카피 작성 (라이터)
블로그 본문 + SNS 홍보문구 + 매물 요약 카드 작성
Step 4 — 산출물 생성 (디자이너)
엑셀 시세표, PPT 매물장, 카드뉴스 이미지 생성
Step 5 — 검증 게이트
모든 산출물 품질 확인 — 오류 시 자동 수정
Step 6 — 발행 (퍼블리셔)
워드프레스 블로그 발행 + 인스타그램 자동 게시 완료!
입력 1줄 → 7종 산출물 + 2개 플랫폼 자동 발행 = 약 5분

MCP 도구 확장 — AI의 손과 발

MCP(Model Context Protocol)는 AI에게 외부 도구를 연결해주는 기술입니다.
마치 스마트폰에 앱을 설치하듯, AI에게 새로운 능력을 추가합니다.

📧

Gmail

이메일 자동 작성, 검색, 답장

📅

Google Calendar

일정 생성, 빈 시간 찾기

📋

Notion

문서 작성, DB 관리, 보고서 저장

🌐

Playwright

웹 브라우저 자동 조작, SNS 게시

📝

WordPress

블로그 글 작성, 미디어 업로드

🔍

Web Search

실시간 인터넷 검색, 최신 정보

MCP = AI의 앱스토어

MCP 도구를 연결할수록 AI가 할 수 있는 일이 늘어납니다.
이메일 보내기, 일정 잡기, 블로그 발행, SNS 게시 — 전부 AI가 직접 처리합니다.

미래 비전 — 한 문장이면 사업이 돌아간다

스킬이 충분히 진화하면, 여러분이 할 일은 딱 한 줄 입력뿐입니다.
나머지는 AI 팀이 알아서 처리합니다.

"매물접수 힐스테이트영통 전세 5.8억"

7
종 산출물
2
플랫폼 발행
5분
소요 시간

이것이 스킬의 최종 목표입니다.
여러분은 방향만 지시하고, AI가 실행합니다.
당신은 사장님이고, AI는 당신의 팀입니다.

이렇게 하세요

CLAUDE.md 구성 가이드

CLAUDE.md에 꼭 들어가야 할 4가지 필수 섹션입니다. 

# CLAUDE.md — 필수 섹션 4가지

## 1. Hard Rules (절대 규칙)
- plan.md 승인 전 코드 작성 금지
- API 키 하드코딩 금지
- 한국어 주석 필수
- 작업 완료 후 노션 보고서 업로드

## 2. Design Standard (디자인 기준)
- "내가 돈 주고 사겠는가?" 기준
- 텍스트만 나열하면 재작업
- 50대 이상 사용자 기준 가독성

## 3. Security (보안 규칙)
- API 키는 반드시 .env 파일 사용
- 민감 정보 GitHub 커밋 금지
- 키 변경 시 즉시 보고

## 4. Agent Modes (모드 설정)
- "설계" → /architect 모드 전환
- "검토" → /critic 모드 전환
- "QA" → 자체 품질 검증 실행

Hard Rules

"이건 무조건 지켜라"
가장 중요합니다. 이것부터 쓰세요.
예: 승인 없이 실행 금지

Design Standard

"이 수준 이하면 다시 해라"
결과물 품질의 최소 기준선입니다.
예: 구매자 시각 기준 평가

Security

"비밀은 지켜라"
API 키, 비밀번호 유출 방지 규칙입니다.
예: .env 파일로 키 관리

Agent Modes

"이 말 하면 이 모드로 전환해라"
키워드별 동작 모드를 지정합니다.
예: "설계" → plan.md 작성
